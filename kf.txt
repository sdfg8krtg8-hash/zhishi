
适配Vue3+TS+Vite项目，仅规划架构、页面结构、数据字段、图片接口，全程严格遵循简约高级视觉标准，图片位置统一预留，后续手动补充素材，可直接交付开发。

一、项目基础说明

1.1 项目简介

项目名称：知食
项目定位：面向烹饪零基础、不认识食材的新手，打造食材查询、食谱学习网站。核心原则：功能实用+视觉素雅高级，拒绝花哨装饰，以清晰文字为主，图片为辅助元素。
技术栈：Vue3 + TypeScript + Vite + Vue Router + Pinia
开发要求：搭建完整项目框架，划分标准化资源目录，图片接口统一预留、默认使用占位图，后续仅替换素材即可；全程把控布局、留白、排版、组件样式，保证全站视觉协调美观。

1.2 全局视觉规范（强制落地，保障整体美观）

1. 色彩体系
主色：棕褐色，用于标题、按钮、选中状态、重点文字
辅助色：米杏色（页面背景）、浅棕灰（分割线、卡片边框）
文字色：深灰（主标题/核心正文）、中灰（常规内容）、浅灰（备注/贴士）
禁用：高饱和亮色、荧光色、渐变、夸张撞色，整体色调温润统一。

2. 字体规范
统一使用思源黑体+Inter，层级分明：页面大标题、模块标题、正文、备注小字字号逐级递减，行间距宽松，提升阅读美感与舒适度，不使用艺术字体。

3. 布局与留白（美观核心）
全站统一留白标准：页面左右预留充足边距，模块、卡片、组件之间间距均等，杜绝内容拥挤。
卡片样式：纯白底色、细边框、统一圆角，无复杂阴影、浮雕；鼠标悬浮仅轻微加深边框，无缩放、弹跳等浮夸动效。
图文比例：图片区域尺寸统一，不出现大小不一、比例失衡的情况，图文排布工整对称。

4. 适配要求
同时适配PC端、移动端，不同设备下布局自动自适应，按钮、文字、图片比例协调，移动端保证视觉整洁、操作舒适。

二、项目整体目录架构

按功能分层管理，图片资源独立分区，目录规整有序，便于维护与素材补充。

1. public 公共静态目录
存放网站全局图标，预留网站图标位置，图标风格与全站简约风统一。

2. src/assets 资源总目录

◦ images 图片总文件夹（所有图片分类存放，接口统一）

◦ ingredients：食材主图专属目录

◦ recipes：菜品封面图、烹饪步骤图专属目录

◦ icons：导航、分类、功能图标目录

◦ placeholders：全局通用占位图目录（图片未填充时默认展示，样式统一）

◦ styles 全局样式文件夹：统一配置配色、字体、边距、圆角、卡片样式等基础视觉规则，全站样式同源。

3. src/components 公共组件目录

◦ layout 布局组件：顶部导航栏、底部页脚（全站样式统一，视觉风格一致）

◦ common 卡片组件：食材卡片、食谱卡片（尺寸、圆角、边距、样式全局标准化）

◦ ui 通用交互组件：搜索栏、筛选栏（按钮、输入框样式统一，线条简洁）

4. src/router 路由目录：配置页面跳转逻辑

5. src/stores 状态管理目录：管理收藏、数据等全局状态

6. src/types 类型定义目录：定义标准数据结构与图片字段

7. src/data 模拟数据目录：存放文本内容，图片路径字段留空

8. src/views 页面目录：存放所有独立页面，布局、排版严格遵循视觉规范

三、数据结构与图片接口规范

统一数据模型与图片字段，图片区域尺寸、引用规则固定，替换素材不破坏页面美观。

3.1 食材数据模型

基础字段：ID、名称、别名、分类、文字介绍、选购技巧、保存方法、搭配食材、关联食谱ID。
图片字段：食材主图，固定引用assets/images/ingredients目录，图片比例统一（标准竖版/方版）。

3.2 食谱数据模型

基础字段：ID、菜品名、难度、时长、菜系、食材清单、步骤文字、新手贴士。
图片字段

1. 菜品封面图：引用assets/images/recipes目录，统一方形比例，适配所有卡片与详情页头部；

2. 分步操作图：每个步骤独立配图，图片尺寸、比例完全一致，保证步骤区排版整齐。

3.3 通用图片规范

1. 功能/分类图标：统一尺寸、线性简约风格，存放于icons目录；

2. 占位图：全站使用同一款式占位图，风格素雅，和整体设计融合，避免突兀；

3. 所有图片区域预设固定宽高与比例，防止后续素材替换后出现变形、排版错乱。

四、公共组件设计（样式标准化，保障局部美观）

所有复用组件样式、尺寸、间距全局统一，是全站视觉协调的基础。

4.1 导航栏组件

整体横向均分布局，左右中部区域划分清晰；Logo图标预留固定尺寸位，搜索框、功能按钮样式简约、大小统一，线条柔和，和页面风格融为一体。

4.2 页脚组件

居中排版，文字间距均匀，极简设计，不添加多余元素，和顶部导航形成视觉呼应。

4.3 食材卡片组件

固定统一外框尺寸、圆角、边距；上半部分为图片区（固定比例），下半部分为文字区，文字居中对齐；批量排列时行列对齐、间距均等，整体观感整齐有序。

4.4 食谱卡片组件

尺寸、边框、圆角、间距与食材卡片保持一致，实现卡片类组件视觉统一；上方封面图区域比例固定，下方文字标签排布错落有致、层次清晰。

4.5 搜索栏、筛选栏组件

输入框、按钮尺寸统一，横向排布疏密得当；按钮选中状态仅用主色区分，样式简洁，无多余装饰。

五、全站页面框架设计（布局+排版+图片位，重点把控美观）

共6个核心页面，布局规整、留白充足、元素对齐，所有图片位置预设尺寸与比例，框架阶段就保证页面美观，填充素材后无需调整样式。

5.1 首页

整体分区明确，从上至下视觉节奏舒缓；

1. 小白快捷专区：卡片等距排列，行列对齐，图片位比例统一；

2. 食材分类导航：图标+文字组合，图标尺寸一致，排布对称美观；

3. 热门食谱推荐：横向滚动卡片间距均匀，整体画面整洁；
全页留白充足，模块之间用细分割线区分，过渡自然不杂乱。

5.2 食材列表页

顶部筛选栏居中排布，按钮整齐划一；下方食材卡片采用标准网格布局，行列严格对齐，卡片间距完全相等，视觉规整大气，所有图片区域比例统一。

5.3 食材详情页（核心页面）

采用标准左图右文布局，左右区域宽度比例协调；
左侧食材大图区：固定尺寸与比例，居中展示，画面视觉重心稳定；
右侧文字区：段落分段清晰、行间距统一，标题与正文层级明显；
底部关联食谱区：复用标准卡片，延续整体排版风格，页面上下视觉连贯。

5.4 食谱列表页

顶部多条件筛选栏横向排布，分区清晰、按钮疏密合理；下方食谱卡片网格布局，和食材列表页卡片样式、间距保持统一，全站卡片视觉风格闭环。

5.5 食谱详情页（核心页面）

流式自上而下布局，段落、区块间距均匀，阅读动线流畅；

1. 顶部菜品封面大图：居中展示，预设标准比例，占据视觉主区域，大气简洁；

2. 食材清单区：表格线条简约，单元格间距统一，排版工整；

3. 烹饪步骤区：每一步图文组合高度、宽度一致，步骤之间间距均等，所有步骤配图尺寸完全相同，纵向排列整齐；

4. 贴士区、功能按钮区：居中/居左对齐，按钮样式统一，底部视觉收尾干净。

5.6 我的收藏页

分类切换按钮样式统一；收藏食材、收藏食谱板块分区清晰，复用全局标准卡片；空状态区域预留占位图位置，占位图与页面风格融合，整体不空洞、不突兀。

六、路由与功能逻辑

路由关联全部页面，页面跳转逻辑简洁；搜索、筛选、收藏等功能仅保留基础交互样式，按钮点击反馈柔和，不破坏页面整体美感。
